# <<COMPONENTNAME>> — agent guidance

This is a **Python processing component** (`<<COMPONENTFULLNAME>>`) generated by the `edgecommons`
CLI from the `python-processor` template. It runs on the `edgecommons` library and follows the
org-wide conventions in the parent EdgeCommons workspace's `AGENTS.md` — read that first if you have
it available; what follows is scoped to this repo.

## What this component is

A processor **subscribes**, **transforms**, and **forwards**. Each entry of `component.instances[]`
is one route: topic filters, a pipeline of stages, and a target (`local` or `northbound`). Routes are
independent — one thread each — so a slow route cannot stall another.

## Layout

- `main.py` — builds `EdgeCommons` and hands off to `app/<<COMPONENTNAME>>.py`.
- `app/<<COMPONENTNAME>>.py` — the wiring: routes, subscriptions, the bounded queue, the tick,
  publishing, metrics, events.
- `app/pipeline.py` — **where your code goes**: stages, the pipeline, the self-echo guard, the route
  parser. Deliberately does not import `edgecommons` — it is pure, unit-testable logic.
- `config.schema.json` — the config this component itself understands. Keep `additionalProperties:
  false`; a stage you add to `_STAGES` needs a matching schema variant.
- `tests/` — `pytest` tests for the pipeline invariants and the connectivity seam. No broker needed.
- `docs/` — Diátaxis documentation describing the generated component; keep it in sync with the code.

## Non-negotiable invariants (do not remove)

- **The self-echo guard** (`is_self_echo` in `app/pipeline.py`). A route that publishes onto a class
  it also subscribes to will consume its own output, reprocess it, and republish it forever without
  this check — an MQTT broker redelivers a component's own publishes regardless of
  `receive_own_messages(False)`.
- **The identity restamp** (`gg.instance(route.id).new_message(...)` in `_dispatch`). What a route
  publishes is *ours*, not the producer's — without the restamp, downstream self-echo guards cannot
  work either.
- **The bounded queue that drops and counts** (`put_nowait` / `dropped` measure in `_handler`). Never
  make the queue unbounded or swallow a full-queue condition silently.

## Validation expectations

- `python -m pytest` must pass with no broker, no device, and no cloud credentials.
- The org coverage gate is **90% line coverage** (`.github/workflows/ci.yml`'s `coverage` job).
- A stage added to `app/pipeline.py`'s `_STAGES` table needs a matching `config.schema.json` variant
  in the same change — the two are one contract.

## Docs stay in sync with code

Any change that adds/removes a stage, a route shape, a metric, or an event must update the
corresponding page under `docs/` in the same change. Treat stale docs as a defect, not cosmetic drift.

## Building against the library

scaffold time. See `CLAUDE.md` for the local-dev override.
