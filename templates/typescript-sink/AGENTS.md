## What it is

Consumes messages off the bus and delivers each one to a destination, idempotently and verified,
with retry-with-jitter and a full event ladder on failure. Ships with a **local-filesystem
destination** (`src/dest.ts`'s `LocalDestination`) so it runs with nothing to configure; the
intended first change is implementing a real backend. Runs on `HOST` / `GREENGRASS` /
`KUBERNETES` via the standard edgecommons platform/transport model — no platform branching in this
component's own code.

## The seam

`src/dest.ts`'s `Destination` interface (`kind` / `deliver` / `verify`) is the one thing a real
backend implements. Two properties are non-negotiable regardless of backend: delivery is
**idempotent to a stable key**, and it is **verified before the source is released**. Everything
above the seam (`src/app.ts`'s retry ladder, event emission, connectivity reporting) is written
against the interface and does not change when the backend does.

## Validation expectations

- `npm run build` (tsc) and `npm test` (vitest) must stay green; `npm run coverage` enforces the
  org's 90%-line gate (`vitest.config.ts` thresholds).
- The delivery ladder's ordering (deliver → verify → confirm, retry only on a transient
  classification, the event sequence) is a load-bearing invariant (see `docs/explanation.md`) — a
  change that skips verification or misclassifies a failure is a regression, not a simplification.
- `sinkDeliveries.exhausted` is the metric that matters: it must always correspond to a
  `delivery-exhausted` event, never a silent drop.
