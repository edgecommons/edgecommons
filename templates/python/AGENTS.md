## Layout

- `main.py` — builds `EdgeCommons` and hands off to `app/<<COMPONENTNAME>>.py`.
- `app/<<COMPONENTNAME>>.py` — the component. Replace the demo metric/signal/event/command with real
  logic; keep the `instance_connectivity()` seam if this component grows a southbound connection.
- `config.schema.json` — the config this component itself understands (`component.global`, and
  `component.instances[]` once you add any). Keep `additionalProperties: false`.
- `tests/` — `pytest` tests for the command verb and the connectivity provider. No broker needed.
- `docs/` — Diátaxis documentation describing the generated component; keep it in sync with the code
  (see the "docs stay in sync" rule below).

## Validation expectations

- `python -m pytest` must pass with no broker, no device, and no cloud credentials.
- The org coverage gate is **90% line coverage** (`.github/workflows/ci.yml`'s `coverage` job). Add
  tests rather than lowering the gate or excluding testable code.
- Config changes go in `config.schema.json` **and** in the code that reads them — the two are one
  contract; an unknown config key should be rejected, not silently ignored.

## Docs stay in sync with code

Any change that lands, removes, or changes a metric/signal/event/command/config key must update the
corresponding page under `docs/` in the **same** change — `docs/reference/messaging-interface.md` and
`docs/reference/metrics.md` in particular describe exact topics and measure names. Treat stale docs
as a defect, not cosmetic drift.

## Building against the library

`requirements.txt` names the `edgecommons` dependency in the form chosen with `--dep-source` at
scaffold time (`local`: an editable sibling checkout; `registry`/`pinned-rev`: a git dependency). See
`CLAUDE.md` for the local-dev override, and `edgecommons component upgrade` to bump the pin.
