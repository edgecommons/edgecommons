## What this component is

A sink is the last thing standing between data and its destination: it consumes work off the bus,
delivers it outward, verifies what landed, and only then releases the source. Each entry of
`component.instances[]` is one sink: a subscription, a destination, and a retry policy.

## Layout

- `main.py` ‚Äî builds `EdgeCommons` and hands off to `app/<<COMPONENTNAME>>.py`.
- `app/<<COMPONENTNAME>>.py` ‚Äî the wiring: sinks, subscriptions, the bounded queue, the retry loop,
  the event ladder, metrics.
- `app/dest.py` ‚Äî **where your code goes**: the `Destination` abstraction, the local destination, the
  error taxonomy, the retry policy, the config parser. Deliberately does not import `edgecommons` ‚Äî
  it is pure, unit-testable logic.
- `config.schema.json` ‚Äî the config this component itself understands. Keep `additionalProperties:
  false`; a destination type you add to `build_destination()` needs a matching schema variant.
- `tests/` ‚Äî `pytest` tests for the destination/retry/health invariants and the connectivity seam. No
  broker needed.
- `docs/` ‚Äî Di√°taxis documentation describing the generated component; keep it in sync with the code.

## Non-negotiable invariants (do not remove)

- **Deliver to a deterministic, stable key.** `key_for()` ‚Äî the same item must always resolve to the
  same key, so a redelivery overwrites instead of duplicating.
- **Verify before releasing the source.** `Destination.verify()` runs after every `deliver()`, before
  the caller considers the item delivered.
- **Classify failures correctly.** `DeliverError.transient` decides whether retrying can help;
  getting it backwards either burns the retry budget on a permanent failure or gives up on data a
  retry would have saved.
- **The event ladder.** `delivery-started` ‚Üí `delivery-completed` **or** `delivery-failed` (repeated)
  ‚Üí `delivery-exhausted` (critical alarm). Do not skip a rung.

## Validation expectations

- `python -m pytest` must pass with no broker, no device, and no cloud credentials.
- The org coverage gate is **90% line coverage** (`.github/workflows/ci.yml`'s `coverage` job).
- A destination type added to `app/dest.py`'s `build_destination()` needs a matching
  `config.schema.json` variant in the same change ‚Äî the two are one contract.

## Docs stay in sync with code

Any change that adds/removes a destination type, changes the retry defaults, or adds an event/metric
must update the corresponding page under `docs/` in the same change. Treat stale docs as a defect,
not cosmetic drift.
