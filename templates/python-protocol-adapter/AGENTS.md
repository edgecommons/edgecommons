# <<COMPONENTNAME>> — agent guidance

This is a **Python southbound protocol-adapter component** (`<<COMPONENTFULLNAME>>`) generated by the
`edgecommons` CLI from the `python-protocol-adapter` template. It implements the cross-language
**southbound contract** (`docs/SOUTHBOUND.md`) on the `edgecommons` library and follows the org-wide
conventions in the parent EdgeCommons workspace's `AGENTS.md` — read that first if you have it
available; what follows is scoped to this repo.

## What this component is

An adapter connects to devices, reads signals, and publishes them onto the UNS as
`SouthboundSignalUpdate` messages, and serves the standardized `sb/*` command surface plus
`southbound_health`. It ships against an in-process **simulator** (`adapter: sim`) so it runs with no
hardware; a real deployment implements the device seam for its protocol.

## Layout

- `main.py` — builds `EdgeCommons` and hands off to `<<SNAKENAME>>.adapter.App`.
- `<<SNAKENAME>>/device.py` — **the protocol seam**: implement `DeviceSession`/`DeviceBackend` once
  per protocol here. It deliberately imports nothing from `edgecommons` — if it starts to, the seam
  has leaked.
- `<<SNAKENAME>>/adapter.py` — the connect/poll/reconnect worker, per-device `Health`, the write
  allow-list, and instance connectivity. Do not add protocol knowledge here — it belongs in
  `device.py`.
- `<<SNAKENAME>>/metrics.py` — `southbound_health` + the operational-family pattern. The measure
  names in `HEALTH_MEASURES` are the parity anchor a template test asserts against.
- `<<SNAKENAME>>/command_service.py` — the `sb/*` command surface + the three edge-console panels.
- `config.schema.json` — the config this component itself understands. Keep `additionalProperties:
  false`.
- `tests/` — unit tests for every verb/error code/metric measure (no broker/device needed), plus
  `test_live_sim.py` (self-skipping unless `EC_LIVE_SIM` is set — see the how-to guide).
- `docs/` — Diátaxis documentation describing the generated component; keep it in sync with the code.

## Non-negotiable invariants (do not remove)

- **The device seam stays protocol-only.** No EdgeCommons imports in `device.py`.
- **The write allow-list is checked BEFORE any device I/O.** `command_service.py`'s `write()` — do
  not move a write attempt ahead of the `cfg.permits(sid)` check.
- **Every sample carries a quality.** `GOOD`/`BAD`/`UNCERTAIN`, normalized — a failed read publishes
  `BAD`, it is never silently omitted.
- **`southbound_health`'s measure set is exact** (SOUTHBOUND.md §5): `connectionState`,
  `publishLatencyMs`, `pollLatencyMs`, `readErrors`, `staleSignals`, `reconnects`. Do not rename or
  drop one without updating `metrics.py`'s `HEALTH_MEASURES`, the metrics test, and
  `docs/reference/metrics.md` together.
- **Instance routing** (D-EIP-13): `body.instance` optional iff exactly one device is configured;
  otherwise a missing id is `BAD_ARGS`, an unknown id is `NO_SUCH_INSTANCE`.

## Validation expectations

- `python -m pytest` must pass with no broker, no device, and no cloud credentials — `test_live_sim.py`
  self-skips without `EC_LIVE_SIM`.
- The org coverage gate is **90% line coverage** (`.github/workflows/ci.yml`'s `coverage` job).
- A metric family or command verb you add needs a template test asserting its measure names / verb
  behavior, matching the pattern in `tests/test_metrics.py` / `tests/test_commands.py`.

## Docs stay in sync with code

Any change that adds/removes a command verb, a metric family, or a config key must update the
corresponding page under `docs/` in the same change — `docs/reference/messaging-interface.md`,
`docs/reference/metrics.md`, and `docs/reference/data-types.md` describe exact topics, measures, and
the quality model. Treat stale docs as a defect, not cosmetic drift.
