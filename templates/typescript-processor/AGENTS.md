## What it is

Subscribes to messages, runs them through a pipeline of stages, and republishes the result. Ships
with two demo stages (`fieldEquals`, `countPerTick`) wired into one route, so the archetype
(subscribe → bounded queue → pipeline → publish) is observable end to end before you write your own
stage. Runs on `HOST` / `GREENGRASS` / `KUBERNETES` via the standard edgecommons platform/transport
model — no platform branching in this component's own code.

## The seam

`src/proc.ts`'s `Processor` interface is the one thing a real pipeline stage implements. A stage
takes a message and returns zero, one, or many; a stateful stage additionally implements `onTick`.
The pieces around it never change when you add a stage: the self-echo guard, the identity restamp,
the bounded queue, and the stats window live in `src/app.ts` (unit-tested), and the route loop that
drives them lives in `src/runtime.ts` (the thin live-runtime seam, excluded from the coverage gate
and validated by the deploy paths).

## Config location

`component.global.defaults` (fallback `tickMs`/`maxQueue`) and `component.instances[]` (one route
per entry: `id`, `subscribe`, `publishTopic`, `target`, `pipeline`, `maxQueue`, `tickMs`) — see
`config.schema.json` and `docs/reference/configuration.md`. The sibling sections (`tags`,
`hierarchy`, `identity`, `messaging`, `logging`, `metricEmission`, `heartbeat`) are standard
edgecommons sections owned by the canonical schema, not redeclared here.

## Validation expectations

- `npm run build` (tsc) and `npm test` (vitest) must stay green; `npm run coverage` enforces the
  org's 90%-line gate (`vitest.config.ts` thresholds).
- The self-echo guard, the identity restamp, and the bounded-queue drop-and-count behavior are
  load-bearing invariants (see `docs/explanation.md`) — a change that removes any of them is a
  regression, not a simplification.
- A processor uses raw `gg.messaging()`, never the `data()` facade — a processor is payload-agnostic
  and must not impose the `SouthboundSignalUpdate` shape on whatever it republishes.
