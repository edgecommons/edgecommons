## What it is

Connects to devices, polls their signals, and republishes them onto the UNS as
`SouthboundSignalUpdate` messages, plus serves the `sb/*` read/write/browse/reconnect command
family. Ships with an in-process **simulated backend** (`src/device.ts`'s `SimBackend`/
`SimSession`) so it runs with no hardware; the intended first change is replacing that backend with
a real protocol client. Runs on `HOST` / `GREENGRASS` / `KUBERNETES` via the standard edgecommons
platform/transport model — no platform branching in this component's own code.

## The seam

`src/device.ts`'s `DeviceSession`/`DeviceBackend` is the one interface a real protocol
implementation targets. It imports nothing from `@edgecommons/edgecommons` — a backend knows
protocols, not topics, envelopes, or metrics. Everything above the seam is written against the
interface and does not change when the backend does: the unit-tested logic (config parsing, the
reconnect backoff, the control mailbox, health/connectivity, the publish path, and the per-message
control decisions `handleControl`/`serveWhileDown`) lives in `src/app.ts`, the connect/poll/reconnect
supervisor that drives it lives in `src/runtime.ts` (the thin live-runtime seam, excluded from the
coverage gate and validated by the deploy paths + `test/live-sim.test.ts`), `src/commands.ts` is the
command surface, and `src/metrics.ts` the emitters.

## Config location

`component.global` (adapter-wide defaults, `healthThresholds.staleSignalSecs`) and
`component.instances[]` (one entry per device: `id`, `adapter`, `connection`, `pollIntervalMs`,
`writes.allow`) — see `config.schema.json` and `docs/reference/configuration.md`. The sibling
sections (`tags`, `hierarchy`, `identity`, `messaging`, `logging`, `metricEmission`, `heartbeat`)
are standard edgecommons sections owned by the canonical schema, not redeclared here.

## Validation expectations

- `npm run build` (tsc) and `npm test` (vitest) must stay green; `npm run coverage` enforces the
  org's 90%-line gate (`vitest.config.ts` thresholds).
- `test/live-sim.test.ts` self-skips unless `EC_LIVE_SIM` is set — it must stay skipped (not
  failing) in a normal `npm test` run; point it at a real device/simulator once you've replaced
  `src/device.ts`'s simulator.
- Metric measure names (`src/metrics.ts`'s `familyDefs()`) and command error codes
  (`src/commands.ts`) are load-bearing contracts (SOUTHBOUND.md §5/§2.2) — changing them is a
  wire-contract change, not a local refactor.
